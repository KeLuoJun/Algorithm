# 动态规划详解

## 📖 概述

动态规划（Dynamic Programming，简称DP）是一种通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。它适用于有重叠子问题和最优子结构性质的问题。

---

## 🎯 动态规划基本概念

### 核心思想
1. **最优子结构**: 问题的最优解包含子问题的最优解
2. **重叠子问题**: 子问题会被多次求解
3. **状态转移**: 通过状态转移方程来描述问题的递推关系

### 解题步骤
1. **定义状态**: 确定DP数组的含义
2. **状态转移方程**: 找出状态之间的递推关系
3. **初始化**: 确定边界条件
4. **计算顺序**: 确定状态的计算顺序
5. **返回结果**: 从DP数组中得到最终答案

## 📈 一维动态规划

### 经典问题：最长有效括号

#### 问题描述
给你一个只包含 '(' 和 ')' 的字符串，找出最长有效（格式正确且连续）括号子串的长度。

#### 算法思路
- **状态定义**: `dp[i]` 表示以下标 i 字符结尾的最长有效括号的长度
- **状态转移**: 
  - 如果 `s[i] == '('`，则 `dp[i] = 0`
  - 如果 `s[i] == ')'`，需要分情况讨论

#### 代码实现与详解

```python
def longestValidParentheses(s: str) -> int:
    """
    动态规划解法
    时间复杂度: O(n)
    空间复杂度: O(n)
    """
    n = len(s)
    if n <= 1:
        return 0
    
    # dp[i] 表示以下标i字符结尾的最长有效括号的长度
    dp = [0] * n
    max_len = 0
    
    for i in range(1, n):
        if s[i] == ')':
            if s[i-1] == '(':
                # 情况1: ...()
                # 当前位置和前一位置组成一对括号
                dp[i] = (dp[i-2] if i >= 2 else 0) + 2
            elif dp[i-1] > 0:
                # 情况2: ...))
                # 需要检查与当前右括号匹配的左括号位置
                match_pos = i - dp[i-1] - 1
                if match_pos >= 0 and s[match_pos] == '(':
                    # 找到匹配的左括号
                    dp[i] = dp[i-1] + 2 + (dp[match_pos-1] if match_pos > 0 else 0)
        
        max_len = max(max_len, dp[i])
    
    return max_len

# 测试用例
def test_longest_valid_parentheses():
    test_cases = [
        "(()", 2,      # 最长有效括号是 "()"
        ")()())", 4,   # 最长有效括号是 "()()"
        "", 0,         # 空字符串
        "((()))", 6    # 整个字符串都是有效的
    ]
    
    for i in range(0, len(test_cases), 2):
        s = test_cases[i]
        expected = test_cases[i+1]
        result = longestValidParentheses(s)
        print(f"输入: {s}, 期望: {expected}, 结果: {result}")
```

### 经典问题：解码方法

#### 问题描述
一条包含字母 A-Z 的消息通过以下映射进行了编码：'A' -> "1", 'B' -> "2", ..., 'Z' -> "26"。给你一个只含数字的非空字符串 s，请计算并返回解码方法的总数。

#### 代码实现与详解

```python
def numDecodings(s: str) -> int:
    """
    动态规划解法
    dp[i] 表示字符串s[0:i]的解码方法数
    """
    n = len(s)
    if n == 0 or s[0] == '0':
        return 0
    
    # dp[i] 表示s[0:i]的解码方法数
    dp = [0] * (n + 1)
    dp[0] = 1  # 空字符串有1种解码方法
    dp[1] = 1  # 第一个字符（非0）有1种解码方法
    
    for i in range(2, n + 1):
        # 单独解码当前字符
        if s[i-1] != '0':
            dp[i] += dp[i-1]
        
        # 与前一个字符组合解码
        two_digit = int(s[i-2:i])
        if 10 <= two_digit <= 26:
            dp[i] += dp[i-2]
    
    return dp[n]

# 优化空间复杂度的版本
def numDecodingsOptimized(s: str) -> int:
    """
    空间优化版本，只使用两个变量
    """
    n = len(s)
    if n == 0 or s[0] == '0':
        return 0
    
    prev2 = 1  # dp[i-2]
    prev1 = 1  # dp[i-1]
    
    for i in range(2, n + 1):
        current = 0
        
        # 单独解码当前字符
        if s[i-1] != '0':
            current += prev1
        
        # 与前一个字符组合解码
        two_digit = int(s[i-2:i])
        if 10 <= two_digit <= 26:
            current += prev2
        
        prev2 = prev1
        prev1 = current
    
    return prev1
```

## 📊 二维动态规划

### 经典问题：编辑距离

#### 问题描述
给你两个单词 word1 和 word2，请返回将 word1 转换成 word2 所使用的最少操作数。你可以对一个单词进行如下三种操作：插入一个字符、删除一个字符、替换一个字符。

#### 算法思路
- **状态定义**: `dp[i][j]` 表示将 word1 的前 i 个字符转换为 word2 的前 j 个字符所需的最少操作数
- **状态转移**: 根据当前字符是否相等，选择不同的操作

#### 代码实现与详解

```python
def minDistance(word1: str, word2: str) -> int:
    """
    编辑距离 - 动态规划解法
    时间复杂度: O(m*n)
    空间复杂度: O(m*n)
    """
    m, n = len(word1), len(word2)
    
    # dp[i][j] 表示word1前i个字符转换为word2前j个字符的最少操作数
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    # 初始化边界条件
    # word1为空，需要插入j个字符
    for j in range(n + 1):
        dp[0][j] = j
    
    # word2为空，需要删除i个字符
    for i in range(m + 1):
        dp[i][0] = i
    
    # 填充DP表
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i-1] == word2[j-1]:
                # 字符相同，不需要操作
                dp[i][j] = dp[i-1][j-1]
            else:
                # 字符不同，选择代价最小的操作
                dp[i][j] = min(
                    dp[i-1][j] + 1,    # 删除word1[i-1]
                    dp[i][j-1] + 1,    # 插入word2[j-1]
                    dp[i-1][j-1] + 1   # 替换word1[i-1]为word2[j-1]
                )
    
    return dp[m][n]

# 空间优化版本
def minDistanceOptimized(word1: str, word2: str) -> int:
    """
    空间优化版本，只使用一维数组
    """
    m, n = len(word1), len(word2)
    
    # 只保留当前行和上一行
    prev = list(range(n + 1))
    
    for i in range(1, m + 1):
        curr = [i] + [0] * n
        
        for j in range(1, n + 1):
            if word1[i-1] == word2[j-1]:
                curr[j] = prev[j-1]
            else:
                curr[j] = min(
                    prev[j] + 1,      # 删除
                    curr[j-1] + 1,    # 插入
                    prev[j-1] + 1     # 替换
                )
        
        prev = curr
    
    return prev[n]
```

### 经典问题：最小路径和

#### 问题描述
给定一个包含非负整数的 m x n 网格 grid，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

#### 代码实现与详解

```python
def minPathSum(grid: List[List[int]]) -> int:
    """
    最小路径和 - 动态规划解法
    """
    if not grid or not grid[0]:
        return 0
    
    m, n = len(grid), len(grid[0])
    
    # dp[i][j] 表示到达位置(i,j)的最小路径和
    dp = [[0] * n for _ in range(m)]
    
    # 初始化起点
    dp[0][0] = grid[0][0]
    
    # 初始化第一行
    for j in range(1, n):
        dp[0][j] = dp[0][j-1] + grid[0][j]
    
    # 初始化第一列
    for i in range(1, m):
        dp[i][0] = dp[i-1][0] + grid[i][0]
    
    # 填充DP表
    for i in range(1, m):
        for j in range(1, n):
            # 只能从上方或左方到达当前位置
            dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]
    
    return dp[m-1][n-1]

# 原地修改版本（空间优化）
def minPathSumInPlace(grid: List[List[int]]) -> int:
    """
    原地修改，空间复杂度O(1)
    """
    if not grid or not grid[0]:
        return 0
    
    m, n = len(grid), len(grid[0])
    
    # 初始化第一行
    for j in range(1, n):
        grid[0][j] += grid[0][j-1]
    
    # 初始化第一列
    for i in range(1, m):
        grid[i][0] += grid[i-1][0]
    
    # 填充网格
    for i in range(1, m):
        for j in range(1, n):
            grid[i][j] += min(grid[i-1][j], grid[i][j-1])
    
    return grid[m-1][n-1]
```

## 🎯 区间动态规划

### 经典问题：赢家预测（博弈DP）

#### 问题描述
给你一个整数数组 nums。玩家1和玩家2基于这个数组设计了一个游戏。玩家1和玩家2轮流进行自己的回合，玩家1先手。每一回合，玩家从数组的任意一端拿取一个数字。这将导致对手下一回合只能从数组的另一端拿取数字。玩家的得分为拿取的数字之和。当数组中没有剩余数字时，游戏结束。

#### 算法思路
- **状态定义**: `dp[i][j]` 表示在数组 nums[i...j] 中，当前玩家比对手多拿的最大分数
- **状态转移**: 当前玩家可以选择拿 nums[i] 或 nums[j]，然后轮到对手

#### 代码实现与详解

```python
def predictTheWinner(nums: List[int]) -> bool:
    """
    区间DP解法 - 博弈问题
    """
    n = len(nums)
    
    # dp[i][j] 表示在nums[i...j]中，当前玩家比对手多拿的最大分数
    dp = [[0] * n for _ in range(n)]
    
    # 初始化：只有一个数字时，当前玩家拿走它
    for i in range(n):
        dp[i][i] = nums[i]
    
    # 区间长度从2开始
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            
            # 当前玩家选择拿nums[i]，对手在nums[i+1...j]中最优选择
            take_left = nums[i] - dp[i+1][j]
            
            # 当前玩家选择拿nums[j]，对手在nums[i...j-1]中最优选择
            take_right = nums[j] - dp[i][j-1]
            
            # 选择对当前玩家更有利的策略
            dp[i][j] = max(take_left, take_right)
    
    # 如果玩家1的得分优势 >= 0，则玩家1获胜
    return dp[0][n-1] >= 0

# 记忆化搜索版本
def predictTheWinnerMemo(nums: List[int]) -> bool:
    """
    记忆化搜索版本
    """
    n = len(nums)
    total = sum(nums)
    memo = {}
    
    def dfs(i, j):
        """返回在nums[i...j]中，当前玩家能获得的最大分数"""
        if i > j:
            return 0
        if i == j:
            return nums[i]
        
        if (i, j) in memo:
            return memo[(i, j)]
        
        # 选择左端点或右端点
        take_left = nums[i] + min(dfs(i+2, j), dfs(i+1, j-1))
        take_right = nums[j] + min(dfs(i+1, j-1), dfs(i, j-2))
        
        result = max(take_left, take_right)
        memo[(i, j)] = result
        return result
    
    player1_score = dfs(0, n-1)
    player2_score = total - player1_score
    
    return player1_score >= player2_score
```

## 🎒 背包动态规划

### 01背包问题

#### 问题描述
给定一个背包容量为 W，有 n 个物品，每个物品有重量 w[i] 和价值 v[i]。每个物品只能选择一次，求背包能装下的最大价值。

#### 代码实现与详解

```python
def knapsack01(weights, values, capacity):
    """
    01背包问题 - 二维DP解法
    """
    n = len(weights)
    
    # dp[i][w] 表示前i个物品，背包容量为w时的最大价值
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        for w in range(capacity + 1):
            # 不选择第i个物品
            dp[i][w] = dp[i-1][w]
            
            # 如果能装下第i个物品，考虑选择它
            if w >= weights[i-1]:
                dp[i][w] = max(dp[i][w], 
                              dp[i-1][w-weights[i-1]] + values[i-1])
    
    return dp[n][capacity]

def knapsack01Optimized(weights, values, capacity):
    """
    01背包问题 - 一维DP优化版本
    """
    n = len(weights)
    
    # dp[w] 表示背包容量为w时的最大价值
    dp = [0] * (capacity + 1)
    
    for i in range(n):
        # 从后往前遍历，避免重复使用同一物品
        for w in range(capacity, weights[i] - 1, -1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    
    return dp[capacity]

# 完全背包问题
def knapsackComplete(weights, values, capacity):
    """
    完全背包问题 - 每个物品可以使用无限次
    """
    n = len(weights)
    dp = [0] * (capacity + 1)
    
    for i in range(n):
        # 从前往后遍历，允许重复使用同一物品
        for w in range(weights[i], capacity + 1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    
    return dp[capacity]
```

## 🔢 状态压缩动态规划

### 经典问题：火柴拼正方形

#### 问题描述
你将得到一个整数数组 matchsticks，其中 matchsticks[i] 是第 i 根火柴棒的长度。你要用所有的火柴棒拼成一个正方形。你不能折断任何一根火柴棒，但你可以把它们连接起来，且每根火柴棒必须使用一次。

#### 代码实现与详解

```python
def makesquare(matchsticks: List[int]) -> bool:
    """
    状态压缩DP解法
    """
    if not matchsticks:
        return False
    
    total = sum(matchsticks)
    if total % 4 != 0:
        return False
    
    side_length = total // 4
    n = len(matchsticks)
    
    # 预处理：计算每个状态的火柴总长度
    sums = [0] * (1 << n)
    for mask in range(1 << n):
        for i in range(n):
            if mask & (1 << i):
                sums[mask] += matchsticks[i]
    
    # dp[mask] 表示使用mask状态的火柴能否组成若干条完整的边
    dp = [False] * (1 << n)
    dp[0] = True
    
    for mask in range(1 << n):
        if not dp[mask]:
            continue
        
        # 尝试添加新的边
        for submask in range(mask + 1, 1 << n):
            # submask 是 mask 的超集
            if (submask & mask) != mask:
                continue
            
            # 新增的火柴组成的边
            new_edge = submask ^ mask
            
            # 检查新边的长度是否等于side_length
            if sums[new_edge] == side_length:
                dp[submask] = True
    
    return dp[(1 << n) - 1]

# 回溯解法（更直观）
def makesquareBacktrack(matchsticks: List[int]) -> bool:
    """
    回溯解法
    """
    if not matchsticks:
        return False
    
    total = sum(matchsticks)
    if total % 4 != 0:
        return False
    
    side_length = total // 4
    matchsticks.sort(reverse=True)  # 优化：从大到小排序
    
    # 四条边的当前长度
    sides = [0] * 4
    
    def backtrack(index):
        if index == len(matchsticks):
            return all(side == side_length for side in sides)
        
        for i in range(4):
            # 剪枝：如果当前边加上这根火柴超过目标长度，跳过
            if sides[i] + matchsticks[index] > side_length:
                continue
            
            # 剪枝：避免重复计算相同长度的边
            if i > 0 and sides[i] == sides[i-1]:
                continue
            
            sides[i] += matchsticks[index]
            if backtrack(index + 1):
                return True
            sides[i] -= matchsticks[index]
        
        return False
    
    return backtrack(0)
```

## 📚 动态规划总结

### 🎯 解题技巧

1. **状态设计**: 
   - 明确状态的含义
   - 考虑状态的维度
   - 确保状态能够表示子问题

2. **状态转移**:
   - 找出状态之间的递推关系
   - 考虑所有可能的转移方式
   - 注意边界条件的处理

3. **优化策略**:
   - 空间优化：滚动数组、状态压缩
   - 时间优化：记忆化搜索、剪枝
   - 实现优化：循环顺序、数据结构选择

### 🔍 常见DP类型

| DP类型 | 特点 | 典型问题 | 状态转移特点 |
|--------|------|----------|-------------|
| 线性DP | 状态呈线性排列 | 最长上升子序列 | 一维状态转移 |
| 区间DP | 状态表示区间 | 矩阵链乘法 | 区间合并 |
| 树形DP | 在树上进行DP | 树的直径 | 父子节点关系 |
| 状压DP | 用位运算压缩状态 | TSP问题 | 集合状态转移 |
| 数位DP | 按数位进行DP | 数字计数 | 数位约束 |
| 概率DP | 涉及概率计算 | 期望问题 | 概率转移 |

### 💡 学习建议

1. **从简单开始**: 先掌握一维DP，再学习多维DP
2. **理解本质**: 重点理解最优子结构和重叠子问题
3. **多做练习**: 通过大量练习培养状态设计的直觉
4. **总结模板**: 整理常见DP类型的解题模板
5. **优化意识**: 学会分析时空复杂度并进行优化

---

> 💡 **核心思想**: 动态规划的核心是"记忆化"，通过保存子问题的解来避免重复计算。掌握状态设计和状态转移是解决DP问题的关键。
